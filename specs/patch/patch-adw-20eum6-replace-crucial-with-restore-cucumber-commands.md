## Issue Summary
**Original Spec:** specs/issue-194-adw-20eum6-replace-crucial-with-sdlc_planner-replace-crucial-with-regression.md
**Issue:** The build agent correctly renamed headings from `## Run Crucial Scenarios` to `## Run Regression Scenarios` in `.adw/scenarios.md` and `.adw/commands.md`, but incorrectly replaced the cucumber-js command values with `N/A`. The `## Run Scenarios by Tag` commands were also wiped to `N/A`. This caused exit code 127 (command not found) when the scenario proof tried to execute `N/A` as a shell command. The stale scenario proof log at `logs/20eum6-replace-crucial-with/scenario_proof/scenario_proof.md` shows `## @crucial Scenarios` because it was generated by the old code before the rename took effect.
**Solution:** Restore the actual cucumber-js commands in both `.adw/scenarios.md` and `.adw/commands.md`, updating the tag from `@crucial` to `@regression`. The scenario proof heading issue is already fixed in `regressionScenarioProof.ts` (line 66 outputs `## @regression Scenarios`) and will self-correct on next proof generation.
